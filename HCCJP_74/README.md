## 概要

毎月第2金曜日14時からはHCCJPの勉強会！

前回（第73回）は、ネットワールドの後藤さんに「Nested Hyper-V環境を使えば専用機がなくてもAzure Localの検証環境が作れる」というお話をいただきました。今回は、同じ「Nested Hyper-V」という土俵を使いつつ、胡田が挑戦しているのは少し違う方向です——**Azure Localではなく、Nested Hyper-Vの土台づくりから、その上に載せるLinux／Windowsの検証環境まで、まるごとコードで展開してしまう**仕組みをご紹介します。

オンプレミスのマシンにNested Hyper-Vの土台を組み、その上へLinux・WindowsのVMをガリガリ展開する——土台からゲストまでを**コードによる冪等な展開**で一気通貫にこなす、新しいオープンソースのツールをいま開発中です（※趣味）。Nested Hyper-Vを使っているので、ホストがオンプレミスでもAzure上のVMでも同じコードがそのまま動く——いまはまずオンプレミスで構築を進めていますが、そこも地続きにしたいという構想です。

今はAIエージェントに頼めばHyper-Vの構築や仮想マシン構成なども自動で行ってくれますが、このような仕組みを間に挟み、AIエージェントにはskillsでその仕組みを伝える形にしたほうが効率よく安定的に環境構築できると考えています。

さらに、普通のWindowsやLinuxが動作するようになったらAzure Localにも手を出そうと構想しています（※Azure Local展開部分は既存のオープンソースを呼ぶ形になると思います）。

今回はその**中間報告**として設計と使用感をお見せし、「こんな構成で作っているけれど、どう思う？」「こんな機能があったら嬉しい」をコミュニティの皆さんと一緒に考えたい回です。

いつもの一方向のセッションではなく、**みんなでアイデアを出し合ってOSSプロダクトを育てる**参加型の会にしたいと思っています。AI時代の検証環境づくりの在り方を、一緒に提案していきましょう！

### 2. オンプレ × 冪等 × AI — コードで建てる検証環境をOSSでみんなで作りませんか？

**胡田 昌彦**（日本ビジネスシステムズ株式会社 / Microsoft MVP）

オンプレミスのマシンにNested Hyper-Vの土台を組み、その上にLinux／Windowsの検証環境を載せる——この一連の流れを丸ごとコードで冪等に展開できるオープンソースツールとして公開する準備を進めています。本セッションはその中間報告です。

お話しする予定の内容：

- なぜ「コードで建てる検証環境」なのか — Nested構成の課題とモチベーション
- 設計の全体像 — Nested Hyper-Vの土台づくりから、ゲストのLinux／Windows展開まで丸ごとコード化
- 冪等性を保った展開の勘所（何度流しても同じ状態になる）
- 同じコードがどこでも動く — オンプレミスでもAzure上でも、というポータビリティの狙い
- 手動コマンド → AIが扱いやすい形 → スキル化、という運用の進化
- AIに指示を出して環境構築を任せる、AI時代の検証環境像
- **デモ / 使用感のご紹介**

そして本題——皆さんと一緒に考えたいこと：

- この構成、どう思いますか？
- どんな機能があったら嬉しいですか？
- 「自分のこの用途で使いたい」というユースケースはありますか？
